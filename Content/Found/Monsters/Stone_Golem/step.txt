Stone Golem（石头人）— 行为树构建完整步骤

【概述】
角色定位：重型坦克型近战怪物，高血量、低速度、少攻击手段
现有资产：骨骼网格(SKM_Stone_Golem)、骨骼(SK_Stone_Golem_Skeleton)、
          物理资产(PA_Stone_Golem_PhysicsAsset)、三种材质实例、纹理集、
          闲置/行走/奔跑/跳跃(全套)动画(ThirdPerson模板)、
          演示关卡(Stone_Golem.umap)
缺失资产：动画蓝图、Character BP、AIController、Blackboard、BehaviorTree、
          Task、Decorator、攻击动画、攻击蒙太奇
特殊问题：石头人没有攻击动画和受击动画，需要从零创建或复用模板动画

二、Blackboard 设计（BB_StoneGolem）

Key Name          | 类型           | 初始值    | 说明
------------------|----------------|-----------|------------------------------
TargetActor       | Object (Actor) | None      | 攻击目标（玩家）
MoveLocation      | Vector         | (0,0,0)   | 移动目的地
MonsterState      | Enum (UInt8)   | 0 (Idle)  | 行为状态
AttackRange       | Float          | 250.0     | 近战攻击触发距离
StompRange        | Float          | 350.0     | 震地AOE范围
ChaseRange        | Float          | 1000.0    | 追击触发距离
CanAttack         | Bool           | true      | 攻击就绪
IsDead            | Bool           | false     | 是否死亡
HomeLocation      | Vector         | (0,0,0)   | 巡逻原点
PatrolRadius      | Float          | 600.0     | 巡逻半径（范围小，走得慢）
AttackCooldown    | Float          | 3.0       | 攻击冷却（慢但重）
StompCooldown     | Float          | 5.0       | 震地冷却（大招）
HasUsedStomp      | Bool           | false     | 本场战斗是否用过震地

MonsterState 枚举：
  0 = Idle（待机）
  1 = Patrol（巡逻）
  2 = Chase（追击）
  3 = Attack_Stomp（震地攻击）
  4 = Attack_Basic（基础近战）
  5 = Hurt（受击）
  6 = Dead（死亡）

四、攻击动画方案（石头人特有的挑战）

石头人没有现成的攻击动画，有以下解决方案，按推荐度排序：

【方案A：复用跳跃动画做震地】（最简单，快速出效果）
  1. 用 ThirdPersonJump_Start（蓄力下蹲）+ ThirdPerson_Jump（起跳）做攻击前摇
  2. 实际不移动（取消位移）
  3. 在"落地"帧：触发震地特效 + 伤害
  4. 用 ThirdPersonJump_End 做攻击后摇
  5. 合并为 AM_StoneGolem_Attack_Stomp 蒙太奇

【方案B：用 Sequencer 手K攻击动画】
  1. 在 Sequencer 中打开 SKM_Stone_Golem
  2. 手动 Key 两个攻击姿态：
     - 震地: 双手举起 → 砸地 → 恢复站立（约2秒）
     - 挥拳: 右手后摆 → 前挥 → 恢复（约1.5秒）
  3. 导出为 Animation Sequence → 转为 Montage

【方案C：从 Fab/Marketplace 找免费石头人攻击动画】
  搜索 "Stone Golem Attack Animation" → 导入使用

【方案D：纯逻辑攻击（无动画，但不推荐）】
  完全不做动画，只在 Task 中做碰撞检测 + 特效 + 伤害。
  简单但缺少视觉反馈，作为临时方案可接受。

推荐先用方案A快速验证行为树逻辑，后续再用方案B或C打磨。

五、关键 Task 详细设计

【BTT_StoneGolem_Stomp】震地AOE攻击
  事件: Receive Execute AI
  1. 面向 TargetActor
  2. 播放 AM_StoneGolem_Attack_Stomp 蒙太奇
  3. 在 AnimNotify 帧触发：
     a. Niagara: SpawnSystemAtLocation(NS_StoneGolem_Stomp, Pawn.Location)
        - 如果没有 Niagara 资产：用 SpawnEmitter 代替
        - 粒子效果：从脚底扩散的冲击波、碎石飞溅、尘土扬起
     b. Camera Shake: Get Player Camera Manager → Start Camera Shake
        - 震感：大振幅、低频（体现重型怪物）
     c. AOE 伤害检测:
        SphereOverlapActors(Pawn.Location, StompRange)
        - 过滤: 非自身、玩家角色
        - 伤害: StompDamage = 50.0（高伤害）
        - 击退: LaunchCharacter(Target, KnockbackDirection * 500, ...)
          但 LaunchCharacter 对 Character 有效，直接调用
     d. 短暂暂停 Montage 0.2s（"砸地那一瞬"的停顿感）
  4. Montage 结束 → Finish Execute

【BTT_StoneGolem_GroundSlam】基础近战攻击
  事件: Receive Execute AI
  1. 面向 TargetActor
  2. 如果没有攻击蒙太奇：直接用 Timer 驱动攻击节奏
     a. t=0.0s: 设置 bIsAttacking = true（ABP 中触发攻击姿态混合）
     b. t=0.3s: 前摇结束，攻击帧
        - SphereOverlapActors(AttackRange) → BasicAttackDamage (25.0)
        - 命中特效: 碎石粒子
     c. t=0.8s: 后摇结束
     d. t=1.0s: Finish Execute
  3. 如果有蒙太奇：正常播放 Montage + AnimNotify

  这种 Timer 方式的好处是无需任何动画资产就能测试战斗逻辑。

【BTT_StoneGolem_PatrolGetPoint】获取巡逻点
  与 Stickman 逻辑相同。
  区别：PatrolRadius 更小(600)，因为石头人走得慢，范围太大走不完。

六、BP_StoneGolem_Character 关键配置

【组件】
  - SkeletalMesh: SKM_Stone_Golem
  - AnimClass: ABP_StoneGolem（需新建）
  - CapsuleComponent: 默认（可适当调大，体现体型）
  - AIPerceptionComponent: 添加
  - CharacterMovementComponent: 默认

【变量】
  - Health: Float = 500.0（高血量，三只怪物中最耐打）
  - MaxHealth: Float = 500.0
  - BasicAttackDamage: Float = 25.0
  - StompDamage: Float = 50.0
  - MoveSpeed_Patrol: Float = 150.0（很慢）
  - MoveSpeed_Chase: Float = 300.0（追击也不快，但比巡逻快）
  - StompKnockbackForce: Float = 500.0
  - bIsAttacking: Bool = false
  - HurtTriggered: Bool = false

【CharacterMovement 调参】
  - Max Walk Speed: 300.0
  - Max Acceleration: 1024.0（加速慢，迟钝感）
  - Braking Deceleration Walking: 512.0（刹车也慢）
  - Ground Friction: 8.0（高摩擦，体现沉重）
  - Mass: 500.0（高重量，难以被击退）

【AIPerceptionComponent 配置】
  AI Sight Config:
    - Sight Radius: 800.0（短视野：石头人"视力差"）
    - Lose Sight Radius: 1000.0
    - Peripheral Vision Half Angle: 180°（但360°感知 — 石头人不需要转头）
      注意：UE Sight Config 最大半角是180°（即全方向）
  AI Hearing Config:
    - Hearing Range: 600.0（听觉也差）

  这个配置体现"迟钝但一旦发现就追到底"的特性。

【接口实现】
  - BPI_Damageable:
      ApplyDamage(Damage) → Health -= Damage
      → HurtTriggered = true
      → 如果 Health <= 0 → IsDead = true
      → 注意：给石头人加一个 DamageMultiplier 变量 = 0.7
        用伤害减免体现"石头"的坚硬

十、独特设计要点

1.【重型感】石头人的核心特色是"重、慢、狠"：
   - 移动速度是三只怪物中最慢的(150 patrol / 300 chase)
   - 但攻击力最高(25基础/50震地)
   - 血量最高(500)
   - 通过 CharacterMovement 参数（Mass、Friction、Braking）体现重量感

2.【震地AOE】石头人的标志性技能：
   - 首次进入战斗时触发
   - 大范围 AOE 伤害 + 震屏 + 粒子特效
   - CD 长(5s)，每场战斗仅一次（HasUsedStomp）
   - 击中后可以加一个短暂 Stun 效果(Set Custom Time Dilation = 0.2, 持续0.5s)

3.【无动画后备方案】因为缺少攻击动画，设计了 Timer 驱动的纯逻辑攻击。
   这保证了行为树可以先跑起来，动画后续补。

4.【锐利感知 vs 钝感设计冲突】UE 的 Sight 最大半角只能到180°，
   而360°感知需要额外的设置。简化做法：设180°半角 + 短距离(800)，
   实际上覆盖前方半球，已经够用。

5.【可以设计"弱点"】石头人背后可以有一个弱点碰撞体，
   从背后的攻击伤害 x2，增加战斗策略性。但这属于进阶设计，先不做。
